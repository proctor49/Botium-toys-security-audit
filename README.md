## 📋 Project Overview

This repository documents a comprehensive internal security audit conducted for **Botium Toys**, a fictional small U.S. toy company with a growing online presence. The audit was performed as part of the **Google Cybersecurity Certificate** program and simulates a real-world security assessment scenario.

The audit evaluates Botium Toys' security posture across administrative, technical, and physical control domains, assesses compliance with industry standards (PCI DSS, GDPR, SOC), and produces prioritized remediation recommendations.

---

## 🚨 Key Findings at a Glance

| Severity | Finding |
|----------|---------|
| 🔴 High | No encryption for customer credit card data (PCI DSS violation) |
| 🔴 High | All employees have access to sensitive PII/SPII and cardholder data |
| 🔴 High | No disaster recovery plan or critical data backups |
| 🔴 High | Least privilege and separation of duties not implemented |
| 🟠 Medium | No intrusion detection system (IDS) deployed |
| 🟠 Medium | Weak password policy; no centralized password management |
| 🟠 Medium | No scheduled maintenance plan for legacy systems |
| 🟢 Low | Firewall configured with appropriate rule sets |
| 🟢 Low | Antivirus software installed and actively monitored |
| 🟢 Low | Physical security adequate (CCTV, locks, fire systems) |
